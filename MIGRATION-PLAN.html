WordPress to Static Site Migration Plan

Project: ajaykmadan.in
Target: Astro static site + Decap CMS + Netlify
Project Location: /Users/ajaymadan/Documents/GitHub/Ajay-Professional-Website

Architecture Overview

Content edited via Cursor IDE or Decap CMS → stored in Markdown files in Git → Astro builds → Netlify deploys → Live site with Google Analytics and Contact Form.

Phase 1: Prerequisites and Content Export

Step 1.1: Export content from WordPress

  1. Log into WordPress admin (ajaykmadan.in/wp-admin)
  2. Go to Tools > Export
  3. Select All content
  4. Download the XML file
  5. Save it in export/ folder

Step 1.2: Export media

Use Tools > Export for media, or a plugin like All-in-One WP Migration. Alternatively, manually download images from Media Library.

Step 1.3: Record site structure

Document: Pages (Home, Projects, Blog, About, Contact), Navigation, Hero copy, Testimonial (Chris Sigala), Logo, Tagline.

Phase 2: Astro Project Setup

Status: Completed. Project already created with full structure.

Phase 3: Convert WordPress Content to Markdown

Step 3.1: Run conversion tool

cd /Users/ajaymadan/Documents/GitHub/Ajay-Professional-Website
npx wordpress-export-to-markdown

Specify output folder as src/content/blog/ for posts.

Step 3.2: Configure for Astro

Ensure frontmatter has: title, pubDate, description. Images go to public/images/.

Step 3.3: Manual content

Create pages for Key Projects, Workflow, Commercials, About. Testimonial in src/data/testimonials.json.

Phase 4: Decap CMS Integration

Status: Completed. Admin at /admin, config in public/admin/config.yml.

After Netlify deploy: Enable Identity, enable Git Gateway, add yourself as user.

Phase 5: Recreate Layout and Pages

Status: Completed. Header, Footer, Homepage, Projects, Blog, About, Contact.

Phase 6: Contact Form

Sign up at formspree.io, create form, add PUBLIC_FORMSPREE_ENDPOINT to .env.

Phase 7: Google Analytics (GA4)

Create GA4 property, get Measurement ID, add PUBLIC_GA_MEASUREMENT_ID to .env and Netlify.

Phase 8: SEO and Redirects

Sitemap auto-generated. Add redirects in public/_redirects for old WordPress URLs.

Phase 9: Deployment to Netlify

  1. Push to GitHub: git init, git add ., git commit, git remote add, git push
  2. Netlify: Import from Git, build command npm run build, publish dist
  3. Enable Identity and Git Gateway
  4. Add custom domain ajaykmadan.in

Phase 10: Cutover and Validation

Pre-launch checklist: pages load, nav works, form submits, GA tracks, Decap works, mobile OK, redirects OK. Then DNS cutover. Post-launch: monitor GA, Search Console, cancel WordPress hosting.
